# Understanding Checks: Deep Agents

## Check Questions
1. What makes a deep agent different from the ReAct agent you built earlier? What changes when agents run for hours instead of seconds?
2. Why do deep agents need sandboxes? What does a sandbox provide?
3. Describe the async workflow: how do you launch a deep agent, monitor its progress, intervene if needed, and retrieve results?
4. What would you monitor in a deep agent system? What would trigger an intervention?
5. When is a deep agent architecture appropriate, and when is it overkill?
6. How do the principles from earlier topics (tool design, context engineering, human-in-the-loop, subagents) come together in a deep agent system?

## Completion Criteria
- The learner understands deep agents as long-running, sandboxed, autonomous workflows
- They can explain the role of sandboxes, async patterns, and monitoring
- They can reason about when deep agent architecture is justified
- They see how all Phase 3 concepts connect: agents, tools, context, oversight, subagents, and deep agents form a coherent architecture stack
- They are ready for Phase 4 (deployment and production)
