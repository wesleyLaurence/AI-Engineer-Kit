# Understanding Checks: Subagents and Multi-Agent Systems

## Check Questions
1. What is the difference between a single agent with many tools and an orchestrator with subagents?
2. Name two benefits of using subagents over a single complex agent.
3. Describe two different orchestration patterns (e.g., sequential, parallel, pipeline). When would you use each?
4. Identify a task in your project that could benefit from subagents. How would you decompose it? What would each subagent do?
5. When would adding subagents make things worse instead of better?

## Completion Criteria
- The learner understands subagents as specialized, delegated agents coordinated by an orchestrator
- They can describe multiple orchestration patterns and their tradeoffs
- They can identify tasks that genuinely benefit from decomposition into subagents
- They retain the "start simple" principle -- multi-agent is not the default
- They can reason about the practical costs: context, tokens, latency, debugging complexity
