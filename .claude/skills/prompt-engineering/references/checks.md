# Understanding Checks: Prompt Engineering

## Check Questions
1. What is the difference between a system prompt and a user message? When would you put something in the system prompt vs. the user message?
2. Write a system prompt for [their project use case]. What behavior does it define? What constraints does it set?
3. When would you use few-shot examples in a prompt? Give an example of a task where few-shot examples would help.
4. What is chain of thought prompting? When does it help and when is it unnecessary?
5. In a real application, why would you use prompt templates instead of hardcoded strings?

## Completion Criteria
- The learner can articulate the distinct roles of system prompts and user messages
- The learner has written at least one system prompt that produces reliable, desired behavior
- The learner understands few-shot prompting and can identify tasks where it adds value
- The learner understands chain of thought and when to apply it
- The learner sees prompts as engineered artifacts that get iterated and refined, not one-shot strings

## What to Update
- `learner/progress.md` — mark "Prompt Engineering" complete
- `learner/notes.md` — add observations about their prompt-writing style, areas to revisit, any prompts that need refinement later
- `learner/project.md` — if they wrote project-specific system prompts, note the design decisions
