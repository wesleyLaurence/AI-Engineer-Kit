# Understanding Checks: Working with Coding Agents Effectively

## Check Questions
1. What makes a good instruction to a coding agent? What makes a bad one? Give an example of each.
2. When you receive code from an agent, what do you check before accepting it?
3. Name a situation where you should let the agent drive, and one where you should take manual control.
4. If the agent writes something wrong, how do you correct it effectively?
5. How can you use a coding agent as a learning tool, not just a code generator?

## Completion Criteria
- The learner can distinguish between vague and specific instructions, with examples
- The learner understands they must review agent output — architecture and correctness, not just syntax
- The learner can identify scenarios for agent autonomy vs. manual intervention
- The learner has a mental model for debugging agent mistakes (specific feedback, not just "fix it")
- The learner sees the agent as a tool to interrogate and learn from, not just to generate code
