# Understanding Checks: Agentic Workflows

## Check Questions
1. What is the difference between a chain and an agent? Give a concrete example of each.
2. When would you use a router instead of an agent?
3. Name a task from your project that could be solved with a chain. Now name one that genuinely needs an agent. What makes them different?
4. What are the costs of using an agent when a simpler architecture would work?
5. If someone tells you "we should make this an agent," what questions would you ask to evaluate that decision?

## Completion Criteria
- The learner can clearly distinguish between simple prompts, chains, routers, and agents
- They can identify which architecture level fits a given task, with reasoning
- They understand the costs of agency: tokens, latency, predictability, debuggability
- They default to "start simple, upgrade when justified" rather than "agents for everything"
